# 里程碑邮件通知

在工作流程的关键节点自动发送邮件通知用户进度。

---

## 触发条件

**仅此 3 种情况发送邮件：**

| 场景 | 说明 | 示例 |
|------|------|------|
| ✅ **任务结束** | 整个任务或主要阶段完成 | 功能开发完成、代码迁移完成 |
| ✅ **遇到问题** | 技术难题、阻塞无法继续 | API 文档缺失、环境配置问题 |
| ✅ **需要确认** | 方向选择、方案决策需要用户拍板 | 技术选型、功能优先级 |

**不要发送邮件：**
- ❌ 仅完成几个子任务
- ❌ 定时进度报告
- ❌ 正常进行中

---

## 邮件生成指南

**opencode 直接生成邮件内容，无需脚本。**

### 邮件生成步骤

1. **判断是否达到触发条件**
   - 检查已完成任务数量
   - 检查执行时长
   - 检查是否有阻塞

2. **直接编写邮件内容**
   - 使用下方模板
   - 填入实际进度信息
   - 保持简洁清晰

3. **调用 email-sender 发送**
   - 使用 `email-sender` skill
   - 传入主题和内容
   - 确认发送成功

### 示例 4：阻塞通知邮件

## 最佳实践

### ✅ 应该做：

1. **准时通知** - 每 10 分钟/每任务完成/全部完成时立即发送
2. **内容清晰** - 使用列表、进度百分比，一目了然
3. **可操作** - 明确下一步计划和需要的决策
4. **简洁** - 控制在 300 字以内，重点突出
5. **一致性** - 使用统一的邮件模板和格式
6. **记录时间** - 记录上次发送邮件时间，避免重复

### ❌ 不应该做：

1. **遗漏通知** - 达到触发条件却不发送邮件
2. **冗长** - 邮件超过 500 字
3. **模糊** - 不说清楚下一步做什么
4. **遗漏关键信息** - 不说明已用时长和进度百分比
5. **重复发送** - 同一事件不要重复发送（如 10 分钟和单任务完成同时触发时，合并为一条）

## 邮件发送检查清单

发送邮件前确认：

- [ ] 已达到触发条件（10 分钟/单任务完成/全部完成/阻塞）
- [ ] 使用正确的模板（定时进度/单任务完成/全部完成/阻塞）
- [ ] 填写所有 `{}` 占位符
- [ ] 检查进度百分比计算正确
- [ ] 已用时长准确
- [ ] 下一步计划具体明确
- [ ] 如有决策事项，提供推荐方案
- [ ] 邮件长度 < 500 字
- [ ] 避免重复发送（同一事件只发一次）

## 与 email-sender 技能配合

**完整流程示例**：

```
1. 判断触发条件 → 已完成 4 个研究任务

2. 生成邮件内容 → 使用里程碑模板填写内容

3. 调用 email-sender → 
   "发送邮件到 user@example.com，主题是'[里程碑完成] 用户评价功能 - 研究阶段'，
   内容是：[邮件正文]"

4. 记录已发送 → 在待办中记录"已发送里程碑邮件"，避免重复
```

**关键点**：
- opencode 直接编写邮件，不需要脚本
- 使用模板保持格式一致
- 发送前快速检查内容准确性
- 发送后记录，避免重复发送
