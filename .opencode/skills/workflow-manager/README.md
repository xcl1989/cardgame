# Workflow Manager Skill

智能工作流程管理 skill，让 opencode 自主判断并行任务并分配子代理。

## 核心功能

- **任务分解**: 将复杂任务拆分为可管理的步骤
- **并行决策**: 自动识别可并行执行的任务
- **子代理分配**: 智能启动 `task` 子代理处理独立任务
- **进度追踪**: 实时更新任务状态，保持可见性
- **质量检查**: 在关键节点验证工作质量
- **流程模板**: 预定义常见工作流程
- **邮件通知**: 完成里程碑时自动发送邮件报告进度

## 子代理分配逻辑

当遇到复杂任务时，opencode 会自动：

1. **分析任务结构** - 识别所有子任务和依赖关系
2. **评估并行化可能性** - 使用决策矩阵评分（5-25 分）
3. **选择子代理类型** - `general` 用于研究/编码，`explore` 用于搜索/探索
4. **并行启动子代理** - 评分≥19 分时启动多个子代理
5. **整合结果** - 汇总所有子代理的输出，继续主任务

### 何时使用子代理

✅ **使用子代理**（评分≥19）:
- 3+ 个独立的研究/探索任务
- 多个文件搜索可以同时进行
- 分析多个独立数据集
- 需要了解多个不相关的代码区域
- 预计节省时间>10 分钟

❌ **不使用子代理**（评分≤10）:
- 任务之间有强依赖关系
- 简单操作（<2 分钟）
- 需要与用户持续交互
- 需要看到之前的对话上下文

### 决策评分速查

| 因素 | 1 分 | 3 分 | 5 分 |
|------|-----|------|-----|
| **独立性** | 全依赖 | 部分独立 | 大部分独立 |
| **数量** | 1-2 个 | 3-4 个 | 5+ 个 |
| **复杂度** | 都简单 | 混合 | 都复杂 |
| **总时间** | <10 分钟 | 10-30 分钟 | >30 分钟 |

## 邮件通知

### 触发条件

**仅此 3 种情况发送邮件：**

| 场景 | 说明 |
|------|------|
| ✅ **任务结束** | 整个任务或主要阶段完成 |
| ✅ **遇到问题** | 技术难题、阻塞无法继续 |
| ✅ **需要确认** | 方向选择、方案决策需要用户拍板 |

**不发送：** 完成子任务、定时报告、正常进行中

### 邮件内容

**任务完成邮件** 包含：
- ✅ 完成说明
- 📊 执行统计（开始时间、总耗时、完成任务数）
- 📍 交付成果

**遇到问题邮件** 包含：
- ⚠️ 问题描述
- 🔍 已尝试的解决方案
- ❓ 需要的帮助
- 💡 建议（如有）

**需要确认邮件** 包含：
- 🤔 需要确认的事项
- 可选方案对比（优缺点、预计耗时）
- 推荐方案及原因

### 示例邮件

**任务完成：**
```
主题：[任务完成] 用户评价功能开发

## ✅ 任务完成：用户评价功能开发

用户评价功能已全部完成，包括前后端实现、测试和文档。

## 📊 执行统计
- 开始时间：14:30
- 总耗时：2 小时 15 分钟
- 完成任务数：9

## 📍 交付成果
1. 后端 API：创建评价、获取评价列表、删除评价
2. 前端组件：评价列表、评价表单、星级评分
3. 测试用例：12 个单元测试，8 个集成测试
4. 文档：API 文档、使用说明
```

**遇到问题：**
```
主题：[遇到问题] Stripe 支付集成 - 需要协助

## ⚠️ 遇到问题

在集成 Stripe 支付时，发现官方文档缺少 webhook 
签名验证的详细说明，无法继续。

## 🔍 已尝试的解决方案
1. 查阅 Stripe 官方文档 - 信息不完整
2. 搜索 Stack Overflow - 找到过时的解决方案

## ❓ 需要的帮助
请确认：是否需要联系 Stripe 技术支持？
还是暂时跳过 webhook 验证，先完成其他功能？

## 💡 建议
建议先继续实现其他支付功能，webhook 验证可以稍后补充。
```

**需要确认：**
```
主题：[需要确认] 用户评价数据存储 - 数据库选型

## 🤔 需要您的确认

用户评价数据存储，发现两种可行方案：

**方案 A：使用现有 PostgreSQL 数据库**
- 优点：与现有系统一致，无需额外依赖
- 缺点：需要添加新表
- 预计耗时：2 小时

**方案 B：使用 MongoDB 存储评价**
- 优点：灵活 schema，适合嵌套结构
- 缺点：需要新增数据库依赖
- 预计耗时：3 小时

**推荐：** 方案 A，避免增加技术栈复杂度。
```

## 使用场景

### 1. 全栈功能开发
**场景**: "添加用户评价功能"
**子代理**: UI 研究、Product 组件查找、认证代码查找、数据库 schema 查找
**节省**: 约 25 分钟

### 2. 代码库迁移
**场景**: "从 Redux 迁移到 Zustand"
**子代理**: Slice 发现、Selector 映射、Dispatch 分析、迁移研究
**节省**: 约 40 分钟

### 3. 多源数据分析
**场景**: "分析 Q4 销售、市场、支持数据"
**子代理**: 销售分析、市场分析、支持分析（并行）
**节省**: 约 20 分钟

### 4. 安全审计
**场景**: "审计代码库安全漏洞"
**子代理**: 认证审查、密钥检测、SQL 风险、XSS 风险、验证分析、最佳实践研究
**节省**: 约 45 分钟

### 5. API 集成
**场景**: "集成 Twilio 发送订单确认"
**子代理**: API 研究、触发点查找、集成模式查找
**节省**: 约 15 分钟

## 时间节省计算

**并行执行时间** = `Max(每个子代理的时间)`
**顺序执行时间** = `Sum(每个子任务的时间)`

**示例**: 4 个独立研究任务，每个 8 分钟
- 顺序：4 × 8 = 32 分钟
- 并行：Max(8, 8, 8, 8) = 8 分钟
- **节省：24 分钟（75% 更快）**

## 最佳实践

### ✅ 应该做：
- 为独立任务启动子代理
- 编写具体、自包含的提示
- 批量启动所有独立子代理
- 等待全部完成后综合结果

### ❌ 不应该做：
- 并行化有依赖的任务
- 过度使用子代理（简单任务不需要）
- 忘记整合子代理结果
- 启动无明确目的的子代理
